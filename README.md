## Executive Summary

### Vấn Đề — "The Swipe Fatigue Crisis"

Thị trường ứng dụng hẹn hò đang đối mặt với một nghịch lý: **hàng triệu người dùng nhưng ngày càng cô đơn hơn**.

| Thống kê đau lòng | Con số |
|---|---|
| Thời gian trung bình mỗi swipe | **< 1 giây** |
| Tỷ lệ match dẫn đến cuộc trò chuyện thực sự | **< 30%** |
| Tỷ lệ cuộc trò chuyện "chết yểu" trong 24h | **> 60%** |
| Người dùng cảm thấy "kiệt sức" khi sử dụng | **77%** |

**Nguyên nhân gốc rễ:**
- **Profile Paralysis**: Người dùng không biết thể hiện bản thân
- **Conversation Dead-ends**: Không biết nói gì sau "Hi"
- **Random Matching**: Thuật toán dựa trên hình ảnh bề ngoài
- **Text-first Culture**: Thiếu tương tác thực sự trước khi gặp

## Tính Năng Nổi Bật

### Blind Video Dating — The Show Format

Trải nghiệm hẹn hò như một **mini game show**:

```
┌─────────────────────────────────────────────────────────────────┐
│  BLIND DATE SESSION                                              │
│  ─────────────────────────────────────────────────────────────  │
│                                                                  │
│  ┌────────────────┐        ┌────────────────┐                   │
│  │   ████████     │   AI   │     ████████   │                   │
│  │   ████████     │  HOST  │     ████████   │                   │
│  │   ████████     │        │     ████████   │                   │
│  │   (BLUR 80%)   │        │   (BLUR 80%)   │                   │
│  └────────────────┘        └────────────────┘                   │
│                                                                  │
│  AI: "Cả hai đều yêu thích coffee! Bạn thích cà phê như         │
│       thế nào? Đen đắng hay sữa ngọt ngào?"                     │
│                                                                  │
│  Session: 2:45 | Topics: 3 | Blurring: 65%                      │
└─────────────────────────────────────────────────────────────────┘
```

**Cơ chế hoạt động:**
1. **Queue vào hàng đợi** với Intent Mode (Date/Study/Friend)
2. **AI Match** dựa trên vector embedding compatibility
3. **Video session** bắt đầu với **100% blur**
4. **AI Host** gợi ý chủ đề dựa trên profile cả hai
5. **Blur giảm dần** khi conversation engaging
6. **Reveal hoàn toàn** khi đạt connection threshold

### Smart Swipe với AI Compatibility Score

Không chỉ swipe — **swipe có ý nghĩa**:

| Tính năng | Mô tả |
|---|---|
| **Super Like + Message** | Gửi lời nhắn độc đáo kèm Super Like (Hinge-style) |
| **AI-ranked Recommendations** | Profile được sắp xếp theo cosine similarity của embedding |
| **Intent Mode Filtering** | DATE / STUDY / FRIEND — match đúng mục đích |
| **Location + Preference Filters** | Hard filters trước, AI ranking sau |

### Profile Doctor — AI Profile Coach

AI đóng vai "bác sĩ" phân tích và cải thiện profile:

- **Score** (0-100): Đánh giá tổng thể hấp dẫn
- **Roast**: Nhận xét thẳng thắn (kiểu Gen-Z)
- **Advice**: Gợi ý cải thiện cụ thể
- **What to Add**: Bio keywords, photos suggestions

### Technology Stack — Được Chọn Vì Business Goals

| Thành phần | Công nghệ | Lý do chọn (Business-first) |
|---|---|---|
| **Frontend** | Next.js 16 + React 19 | Server Components cho SEO tốt, tối ưu First Load |
| **Animation** | Framer Motion | Swipe gestures mượt mà như Tinder |
| **Backend** | NestJS 11 | Modular architecture cho team scale, type-safe |
| **Real-time** | Socket.io | Video dating cần latency < 100ms |
| **Database** | PostgreSQL + pgvector | Vector search cho AI matching không cần infra phức tạp |
| **AI Engine** | Google Gemini | Embedding 768-dim, multimodal (audio analysis), chi phí hiệu quả |
| **Cache** | Redis (ioredis) | Session state cho video dating queue |
| **Media** | Cloudinary | Auto-optimize images, CDN global |
| **Auth** | JWT + bcrypt | Stateless auth cho horizontal scale |

### Database Schema Highlights

```sql
-- Vector search cho semantic matching (pgvector)
SELECT * FROM user_profiles
WHERE 1=1
  AND gender = 'FEMALE'                                    -- Hard filter
  AND intent_mode = 'DATE'                                 -- Hard filter  
ORDER BY bio_embedding <=> $query_embedding                -- Cosine similarity
LIMIT 10;

-- Hybrid: SQL filters FIRST, then vector ranking
```

## Quick Start

```bash
# Clone & Setup
git clone https://github.com/your-org/peerzee-fullstack.git
cd peerzee-fullstack

# Start với Docker (đề xuất)
docker-compose up -d

# Hoặc chạy manual
cd peerzee-backend && npm install && npm run start:dev
cd peerzee-frontend && npm install && npm run dev
```

**Environment Variables cần thiết:**
```env
# Backend
DATABASE_URL=postgresql://user:pass@localhost:5432/peerzee
GEMINI_API_KEY=your-gemini-key
JWT_SECRET=your-secret
CLOUDINARY_URL=cloudinary://...
REDIS_URL=redis://localhost:6379

# Frontend  
NEXT_PUBLIC_API_URL=http://localhost:3000
NEXT_PUBLIC_SOCKET_URL=http://localhost:3000
```
